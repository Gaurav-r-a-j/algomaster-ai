# Flowcharts

A **flowchart** is a visual diagram that represents an algorithm or process using standardized symbols. It helps you plan, visualize, and communicate logic before writing code.

## Why Use Flowcharts?

| Benefit | Description |
|---------|-------------|
| **Visual Thinking** | See the logic before coding |
| **Communication** | Share ideas with non-programmers |
| **Debugging** | Spot logical errors early |
| **Documentation** | Explain how code works |
| **Planning** | Design before implementation |

---

## Standard Flowchart Symbols

| Symbol | Name | Purpose | Example |
|--------|------|---------|---------|
| ⬭ | **Oval** | Start / End | "Start", "End" |
| ▱ | **Parallelogram** | Input / Output | "Read x", "Print result" |
| ▭ | **Rectangle** | Process | "sum = a + b" |
| ◇ | **Diamond** | Decision | "Is x > 0?" |
| → | **Arrow** | Flow direction | Connects symbols |
| ⬡ | **Hexagon** | Preparation | "Set count = 0" |

## Example 1: Check if Number is Even or Odd

### Flowchart Structure

```
    ┌─────────────┐
    │    START    │
    └──────┬──────┘
           ↓
    ╱‾‾‾‾‾‾‾‾‾‾‾‾╲
   ╱  Input: n    ╲
   ╲______________╱
           ↓
       ◇───────────◇
      ╱ n % 2 == 0? ╲
     ╱               ╲
    Yes              No
     ↓                ↓
┌─────────┐    ┌─────────┐
│ "Even"  │    │  "Odd"  │
└────┬────┘    └────┬────┘
     └──────┬───────┘
            ↓
    ┌─────────────┐
    │     END     │
    └─────────────┘
```

### Pseudocode

```text
START
    INPUT n
    IF n % 2 == 0 THEN
        PRINT "Even"
    ELSE
        PRINT "Odd"
    ENDIF
END
```

### Python Code

```python
n = int(input("Enter a number: "))
if n % 2 == 0:
    print("Even")
else:
    print("Odd")
```

## Example 2: Find Maximum of Three Numbers

### Flowchart Structure

```
        ┌─────────────┐
        │    START    │
        └──────┬──────┘
               ↓
        ╱‾‾‾‾‾‾‾‾‾‾‾‾╲
       ╱ Input: a,b,c  ╲
       ╲______________╱
               ↓
           ◇───────◇
          ╱ a > b?  ╲
         Yes        No
          ↓          ↓
      ◇───────◇  ◇───────◇
     ╱ a > c?  ╲╱ b > c?  ╲
    Yes    No  Yes    No
     ↓      ↓   ↓      ↓
   max=a max=c max=b max=c
     └──────┬───┴──────┘
            ↓
        ┌─────────┐
        │ PRINT   │
        │   max   │
        └────┬────┘
             ↓
        ┌─────────────┐
        │     END     │
        └─────────────┘
```

### Python Code

```python
a, b, c = 5, 9, 3

if a > b:
    if a > c:
        max_val = a
    else:
        max_val = c
else:
    if b > c:
        max_val = b
    else:
        max_val = c

print(f"Maximum: {max_val}")
```

## Example 3: Loop - Sum of N Numbers

### Flowchart Structure

```
    ┌─────────────┐
    │    START    │
    └──────┬──────┘
           ↓
    ╱‾‾‾‾‾‾‾‾‾‾‾‾╲
   ╱  Input: n    ╲
   ╲______________╱
           ↓
    ┌─────────────┐
    │  sum = 0    │
    │  i = 1      │
    └──────┬──────┘
           ↓
       ◇───────────◇ ←──────┐
      ╱   i <= n?   ╲       │
     Yes             No     │
      ↓               ↓     │
┌───────────┐  ┌──────────┐ │
│ sum += i  │  │  PRINT   │ │
│  i += 1   │  │   sum    │ │
└─────┬─────┘  └────┬─────┘ │
      └─────────────┘       │
              ↓             │
         ┌─────────────┐    │
         │     END     │    │
         └─────────────┘
```

### Pseudocode

```text
START
    INPUT n
    SET sum = 0
    SET i = 1
    
    WHILE i <= n DO
        sum = sum + i
        i = i + 1
    ENDWHILE
    
    PRINT sum
END
```

### Python Code

```python
n = int(input("Enter n: "))
total = 0
for i in range(1, n + 1):
    total += i
print(f"Sum: {total}")
```

## Decision Structures in Flowcharts

### If-Then-Else

```
         ◇───────────◇
        ╱ condition?  ╲
       Yes            No
        ↓              ↓
    ┌───────┐    ┌───────┐
    │ True  │    │ False │
    │ block │    │ block │
    └───┬───┘    └───┬───┘
        └──────┬─────┘
               ↓
```

### Multiple Conditions (If-Elif-Else)

```
         ◇───────────◇
        ╱ condition1? ╲
       Yes            No
        ↓              ↓
    ┌───────┐    ◇───────────◇
    │ Block │   ╱ condition2? ╲
    │   1   │  Yes            No
    └───────┘   ↓              ↓
            ┌───────┐    ┌───────┐
            │ Block │    │ Else  │
            │   2   │    │ block │
            └───────┘    └───────┘
```

## Loop Structures in Flowcharts

### While Loop

```
    ┌─────────┐
    │ init i  │
    └────┬────┘
         ↓
    ◇─────────◇ ←────┐
   ╱ condition? ╲    │
  Yes           No   │
   ↓             ↓   │
┌─────────┐  ┌─────┐ │
│ Process │  │ End │ │
│ i++     │  └─────┘ │
└────┬────┘          │
     └───────────────┘
```

### For Loop (Count-Controlled)

```
    ┌───────────────┐
    │ i = start     │
    └───────┬───────┘
            ↓
    ◇─────────────◇ ←────┐
   ╱ i <= end?     ╲     │
  Yes              No    │
   ↓                ↓    │
┌─────────┐    ┌─────┐   │
│ Process │    │ End │   │
└────┬────┘    └─────┘   │
     ↓                   │
┌─────────┐              │
│ i = i+1 │──────────────┘
└─────────┘
```

## Best Practices

1. **Start Simple**: Begin with basic logic, add details later
2. **Top to Bottom**: Flow should generally go downward
3. **Left to Right**: Decisions branch left/right, then converge
4. **One Entry, One Exit**: Each symbol should have single entry/exit
5. **Use Connectors**: For complex flows, use numbered circles
6. **Keep It Clean**: Avoid crossing lines when possible

---

## Flowchart to Code Mapping

| Flowchart Element | Code Equivalent |
|-------------------|-----------------|
| Oval (Start) | `def function():` or `main()` |
| Parallelogram (Input) | `input()` or `read()` |
| Rectangle (Process) | Assignment, calculation |
| Diamond (Decision) | `if`, `elif`, `else` |
| Loop back arrow | `while`, `for` |
| Oval (End) | `return` or end of function |

---

## Key Takeaways

1. Flowcharts visualize **logic before coding**
2. Use **standard symbols** for clarity
3. **Diamonds** = Decisions (if/else)
4. **Loops** = Arrows that go back up
5. Always have **clear Start and End**
6. Great for **planning and debugging**
